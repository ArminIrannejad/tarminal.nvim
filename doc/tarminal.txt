1. Introduction                                        *tarminal-introduction*

tarminal.nvim is a terminal runner / REPL integration:

- A shared shell terminal buffer in a full-width split (placed by
  'splitbelow', or forced with `split_position`), reused by runs and shown
  independently in each tab.
- Run the current file with a per-filetype runner command; output is
  separated by run banners.
- File locations in terminal output ("foo.c:12:5:", 'File "foo.py", line
  12') are highlighted and navigable; after a run the cursor is parked on
  the first error. The recognized formats are configurable (`error_patterns`)
  and locations are classified by severity (`error_threshold`).
- Send a visual selection or the cell around the cursor (delimited by
  `cell_marker` lines) to a per-filetype REPL, using bracketed paste.

2. Setup                                                      *tarminal-setup*

Call the setup function, optionally passing overrides for the defaults: >lua

    require("tarminal").setup({
      split_height = 12,
      split_position = "auto",  -- "auto" (follow 'splitbelow') | "bottom" | "top"
      shell = vim.env.SHELL or "/bin/bash",
      follow_run = "focus",     -- "none" | "focus" | "insert"
      follow_repl = "none",
      park_on_error = true,
      cell_marker = "# COMMAND ----------",
      time_runs = true,
      banner = true,            -- "===== RUN[n]: <time> =====" line before each run
      runners = {
        python = "python",
        c = "cc",
        ...
      },
      compilers = {
        "cc", "gcc", "clang", "g++", "clang++", "c++",
        "rustc", "ghc", "swiftc", "gdc", "ocamlopt", "ocamlc",
      },
      repls = {
        python = "ipython",
        ocaml = { cmd = "ocaml", bracketed_paste = false },
        haskell = { cmd = "ghci", bracketed_paste = false,
          block_open = ":{", block_close = ":}" },
        ...
      },
      error_patterns = {         -- error-location formats, tried in order
        { pattern = [[at (%S+) line (%d+)]],  -- your own, added before built-ins
          file = 1, lnum = 2, resolve = false },
        ...
      },
      error_threshold = 0,       -- min severity to act on: 0 note | 1 warn | 2 error
      quickfix = {
        open = true,            -- open quickfix after errors_to_quickfix
        close_terminal = true,  -- close the terminal window as well
      },
    })
<

Options hold settings only — tarminal never creates keymaps. Map keys
yourself to the |:Tarminal| subcommands or the Lua functions; see
|tarminal-mappings|.

Runner and REPL tables are merged by filetype, so each default is
independently overridable. For an interpreted runner, tarminal appends the
file (`python foo.py`). If the command's first program is listed in
`compilers`, tarminal builds the file and runs the result automatically:
`c = "clang -Wpedantic -Wall"` becomes
`clang -Wpedantic -Wall foo.c -o foo && ./foo`. Executable paths and version
suffixes are recognized too (for example, `/usr/bin/clang-17`). For a
compiler the name detection doesn't know, a runner entry can be a table with
an explicit flag instead: `zig = { cmd = "zig build-exe", run_binary = true }`
builds with `-o` and runs the produced binary; `run_binary = false` invokes
the command on the file directly — which may still compile (`zig build-exe
foo.zig`), it just does not produce and run a separate binary. With
`time_runs = true`
(the default), runs are timed only when a `time` binary is installed (the
shell keyword is not relied on); for compiled files only the produced
binary is timed, not compilation.

With `banner = false` a run prints nothing extra — no banner line, and the
previous screen is not pushed into scrollback; output simply appends at the
prompt. Error highlighting and quickfix collection still track the new
run's output; only the view pinning to the banner is skipped.

Everything tarminal types at the shell prompt (run commands, `cd`s, REPL
launches) is prefixed with a space, so it stays out of the shell history
when the shell skips space-prefixed commands: `setopt HIST_IGNORE_SPACE`
(zsh), `HISTCONTROL=ignorespace`/`ignoreboth` (bash); fish does this by
default.

A `repls` entry is the REPL command, or a table with `cmd` and
`bracketed_paste = false` for REPLs that read raw stdin and would see the
bracketed-paste escape sequences as input (the stock ocaml toplevel — the
default entry). With bracketed paste disabled the text is sent unwrapped.

Raw REPLs evaluate each line the moment its newline arrives, breaking
multi-line definitions and indented blocks. Set `block_open` and
`block_close` to the REPL's own block markers (ghci's `:{` and `:}`, its
default entry) to wrap multi-line sends in them so the block is read as one
unit; single-line sends stay unwrapped.

A file location in terminal output is recognized by matching each line
against `error_patterns` in order; `file`, `lnum`, `col` and `type` are
capture indices in the pattern. Your patterns are tried before the
built-ins (which cover `path:line:col` and `File "..."` shapes). By default
a match only counts when the file exists on disk, keeping false positives
out; `resolve = false` trusts the pattern and takes the path as written, so
a location that does not resolve against the terminal's directory (output
from a subfolder, another machine) is still caught. `type` classifies a
location as an error, warning or note — a capture index holding the word,
or a fixed "error"/"warning"/"info". Warnings are highlighted with
|TarminalWarning|, quickfix entries carry the severity, and
`error_threshold` skips anything below it when parking, stepping between
errors (next_error/prev_error) and collecting (set it to 2 to ignore
warnings). An explicit jump_to_error on the current line always jumps,
whatever the severity.

`follow_run` / `follow_repl` control where focus goes after sending to the
terminal: "none" stays in the code window, "focus" moves to the terminal
window in normal mode, "insert" moves there and enters terminal-mode.

With `quickfix = { open = false, close_terminal = false }`,
errors_to_quickfix only populates the quickfix list and reports how many
locations it found, leaving the terminal untouched.

tarminal's terminal buffers are named `tarminal://shell` and
`tarminal://repl:<filetype>` and get the `tarminal` filetype. The plugin
never touches terminal buffers it did not create.

3. Commands                                                *tarminal-commands*

                                                                   *:Tarminal*
:Tarminal [toggle]        Show/hide the shared shell terminal.
:Tarminal run             Save and run the current file. From a non-file
                          buffer, re-run the last run.
:Tarminal exec {cmd}      Run {cmd} in the shared terminal, like emacs'
                          M-x compile. Without {cmd}: always prompt, pre-
                          filled with the last command (<CR> re-runs).
                          Cmdline specials in {cmd} are expanded first —
                          `%`, `%:r`, `#`, `<cword>`, ... (|cmdline-special|;
                          `%:S` shell-quotes) — so `exec make %:r` needs no
                          paths. The prefilled command is already expanded
                          and re-runs verbatim, so a bare exec never re-
                          resolves `%` against the terminal itself. Output
                          gets the same banner and error navigation as a run.
:Tarminal send_cell       Send the cell around the cursor to the REPL.
:'<,'>Tarminal send_selection
                          Send the selected line range to the REPL. The visual
                          mapping preserves exact character/block selections.
:Tarminal jump_to_error   Jump to the file location on the current line
                          (in a terminal buffer).
:Tarminal next_error      Move to the next error location.
:Tarminal prev_error      Move to the previous error location.
:Tarminal errors_to_quickfix
                          Collect error locations into the quickfix list.
                          Closing the terminal window and opening quickfix
                          are controlled by the `quickfix` option.

4. Mappings                                                *tarminal-mappings*

No mappings are ever created by the plugin. Every action is exposed both as
a |:Tarminal| subcommand and as a Lua function on the module
(`require("tarminal").toggle()`, `.run()`, `.send_cell()`,
`.send_selection()`, `.jump_to_error()`, `.next_error()`, `.prev_error()`,
`.errors_to_quickfix()`), so map whatever keys you like: >lua

    local t = require("tarminal")
    vim.keymap.set("n", "<leader>ts", t.toggle, { desc = "Toggle terminal" })
    vim.keymap.set("n", "<leader>ru", t.run, { desc = "Run current file" })
    vim.keymap.set("x", "<leader>ri", t.send_selection, { desc = "Send selection" })
    vim.keymap.set("n", "<leader>rc", t.send_cell, { desc = "Send cell" })
<

tarminal's own terminal buffers get the `tarminal` filetype, so buffer-local
mappings (error navigation, leaving terminal-mode) hook onto the |FileType|
event without touching any other terminal: >lua

    vim.api.nvim_create_autocmd("FileType", {
      pattern = "tarminal",
      callback = function(ev)
        local t = require("tarminal")
        local function map(mode, lhs, rhs)
          vim.keymap.set(mode, lhs, rhs, { buffer = ev.buf })
        end
        map("t", "<Esc><Esc>", [[<C-\><C-n>]])
        map("n", "<CR>", t.jump_to_error)
        map("n", "]e", t.next_error)
        map("n", "[e", t.prev_error)
        map("n", "<C-q>", t.errors_to_quickfix)
      end,
    })
<

5. Highlights                                            *tarminal-highlights*

TarminalError                                                 *TarminalError*
    Error locations in terminal output. Defaults to bold with the
    foreground of DiagnosticError; redefined on |ColorScheme|.

TarminalWarning                                             *TarminalWarning*
    Warning and note locations (see `error_patterns`). Defaults to bold with
    the foreground of DiagnosticWarn; redefined on |ColorScheme|.
